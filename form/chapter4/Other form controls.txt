Other form controls

Multi-line text fields
 - multi-line text 창은 <input> 요소가 아닌 <textarea> 요소를 사용한다. 
 ex) <textarea cols="30" rows="8"></textarea>
 - <textarea>와 <input type="text">와의 차이는 데이터 제출 때 사용하는 줄 띄기를
 유저가 허가받았는가이다. 

 - <textarea>는 또한 닫는 태그를 사용하여 너가 태그 사이에 디폴트값을 넣을 수 있다. 
 반면에 input에서는 value 속성에 넣어야한다. 

 - <textarea>에 html, css, js 등 포함할 경우, 그것은 텍스트 컨텐츠에 랜더링된다.
  
 - 가시적으로, form 컨트롤과 포장은 재조정가능하다. 현재의 브라우저는 드래고 핸들링을 지원하고
 텍스트 구간의 크기를 증가/감소할 수 있다. 

 multi-line 랜더링 조절하기
 - cols: 텍스트 컨트롤의 가시적인 너비를 지정한다. 글자 너비를 평균에 맞추어. 이것은 효율적인
 시작높이가 된다. <textarea>를 조정함으로 변화할 수 있다. 그리고 css를 사용하여 오버라이든해서
 디폴트 값은 아무 값도 지정하지 않았다면 20입니다. 

 - rows: 컨트롤을 위한 가시적인 텍스트의 수를 지정합니다. 이것은 시작 높이이다. <textarea>를
 재조정하는 것을 변화함으로. 그리고 css를 사용 오버라이드해서. 조정이 없다면 디폴트는 2이다. 

 - wrap: 컨트롤이 텍스트를 감싸는 방법을 지정한다. 그 값은 기본값 soft, 이것은 텍스트가 포장 없이
 제출됨을 의미한다. 하지만 브라우저에서 랜더링될 때에는 포장될 것이다. hard는 랜더링에서도 포장되며 제출
 시에도 포장됨을 의미한다. 그리고 ofF는 포장되지 않음을 의마한다.

 textarea 사이즈 재조절하기
 - both : 디폴트 값이며 수직으로 수평으로 재조정 가능함을 의미한다. 
 - horizontal : 수평적으로 조정 가능하다. 
 - vertical : 수직적으로 조정 가능하다. 
 - none : 크기조정이 불가능하다. 
 - block and inline : 블록 또는 인라인 방향으로만 크기 조정을 허용하는 실험값

 묘사를 위한

Drop-down 컨트롤
 - 드롭다운 컨트롤은 단순한 유저가 많은 다른 옵션으로부터 선택하게 하는 것이다. 유저의 인터페이스에
 충분한 공간이 없다면. HTML은 두 form을 드롭다운을 가진다: select box와 autocomplete box이다
 두 개 다 동일한 상호작용을 가진다. 그 컨트롤은 활성활 될때, 그 브라우저는 유저가 선택할 수 있는 값 리스트
 를 보여준다. 

 Select box
 - 단순한 select box는 <select> 요소를 하나 혹은 그 이상의 <option> 요소 만들면 된다. 

 - Basic : 필요로 하다면, 선택 박스를 위한 기본값은 selected 속성을 사용함으로 나타낼 수 있다. 
 <option> 요소에. 이 옵션은 페이지를 불러올 떄 이전에 선택되어진다. 

 - Using optgroup : <option> 요소는 <optgroup>안에 품어질 수 있다. 가시적으로 값의 그룹을
 만들기 위하여
 : <optgroup> 요소에, label 요소의 값은 그 품어진 옵션들 앞에 나타내진다. 그 브라우저는 보통 그들을
 옵션들간에 나누어 보여준다. 그래서 그들은 실제 옵션에 대한 혼란을 줄일 수 있다. 

 - Using the value attribute : option 요소가 value 속성을 명확히 한다면, 그 값은 form 제출될 떄
 제출된다. value 속성이 생략된다면, 위의 예제와 같이 <option> 요소의 컨텐츠를 사용한다. 그래서 value 속성은
 필요로 되지 않는다. 하지만 너가 보다 짧은 문구를 사용하길 원하거나 다른 값을 서버에 보낼 수 있다. 
 : 기본적으로, 선택 박스의 높이는 단일 값을 보여주기에 충분하다. 그 선택적인 size 속성은 많은 옵션이 그 선택에 focus
 되지 않을 때 어떻게할지를 제공해준다. 

 - Multiple choice select box : 기본적으로, 선택상자는 오직 사용자가 단일 값만 선택할 수 있다. multiple 속성은
 <select> 요소에 추가함으로, 너는 사용자가 몇몇 값을 선택할 수 있다. 기본적인 매카닉을 사용함으로

 : 다중 선택 선택 상자의 경우에는, 너는 선택상자가 드롭다운 컨텐츠만큼 크기 않은 것을 알 수 있다. 
 대신에, 모든 값은 한 리스트 안에 있으며 추가적인 size 속성은 위젯의 높이를 조절할 수 있다. 

 : 모든 브라우저가 <select> 요소가 multiple 속성을 지원한다. 

 Autocomplete box 
 - 너는 제안할 수 있다. 자동적으로 완성된 값들을 form 위젯이 <datalist> 요소를 사용하기 위해
  자식 <option> 요소에 특정 값을 보여줌으로써. 그 <datalist>는 id를 뿌여할 필요가 있다. 

 - 그 데이터 리스트는 그 때 <input> 요소에서 뜨는 것이다. list 속성을 사용하면, 그 값은 데이터
 리스트를 엮어주는 값들이다. 
 
 - 데이터 리스트는 form 위젯에 소속되면, 그것의 옵션은 자동 완성 텍스트로 사용된다. 특징적으로는
 이것은 사용자가 드롭다운박스 input에 타이핑하는 것에 대해 가능한 매칭으로 나타난다. 

 - Basic

 - Datalist support and fallbacks
 : 대부분 모든 브라우저는 데이터리스트를 지원하지만 너가 오래된 브라우저를 IE같은 것을 지원하고 싶다면, fallback같은
 트릭을 사용해라

 : fallback을 사용하는 경우, 서버에서 <input>과 <select> 모두 사용되는지 확인하세요. 

 - 덜 중요한 데이터리스트 사용법
 : HTML 공식문서에 따르면, list 속성은 <datalist> 요소가 어떤 종류의 위젯도 사용할 수 있다. 
 이것은 몇몇 사용자가 명확하지 않은 일부 용도로 사용한다. 

다른 form 특징
 - meters and progress bars
 : meters and progress bars는 수를 시각적으로 나타낸다. 

 - Progress
 : progress bar는 시간경과에 따라 지정된 max 속성으로 변화하는 값이다. 바와 같은 것은
 <progress> 요소에 사용하여 생성된다. 

 : 이것은 설문지 질문 충족량이나 나 전체 파일 다운로드 퍼센테지와 같은 진행 보고를 구현하기 위한 것이다. 

 - Meter
 : 미터 바는 max와 min 값을 구분된 범위에 고정된 값을 나타낸다. 이 값은 시각적으로 바로 랜더링된다. 
 그리고 바가 보여지는 방법을 알 수 있다. 우리는 그 값을 다른 값과 비교할 수 있다. 

    - low, high 값은 3파트로 나뉜다. 
    : 가장 낮은 파트는 min과 low 값이다. 포함한다. 
    : 중간 파트는 low와 high 값이다. 독점적인
    : 가장 큰 파트는 high와 max 값이다. 포함하는.

    - optimum 값은 <meter> 요소에 대한 optimum 값에 대해 정의한다. 그것은 범위에 따라 변화한다. 
    : optimum 값이 범위보다 낮으면, 낮은 범위는 우선시되는 파트로 고려된다. 그 중간값은 평균으로 고려되고
    가장 높은 범위는 최악으로 고려된다. 
    : optimum 값이 중간이면, 낮은 범위는 평균으로, 중간범위는 선호하는 부분으로, 높은 범위는 평균으로 고려한다. 
    : optimum 값이 범위보다 높으면, 낮은 범위는 최악으로, 중간 범위는 평균으로, 높은 범위는 선호하는 부분으로 고려된다. 

    모든 브라우저는 <meter> 요소가 미터 바의 색을 변화하는 값을 사용하도록한다. 
    : 선호되는 범위는 초록색
    : 평균 범위는 노란색
    : 위험한 구역은 빨간색
